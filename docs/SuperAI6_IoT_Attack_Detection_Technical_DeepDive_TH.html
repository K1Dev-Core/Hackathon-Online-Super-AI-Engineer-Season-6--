1. ภาพรวม

โจทย์เป็น anomaly detection แบบ Normal-only: train มีข้อมูล Normal เป็นหลัก แต่ไม่มี attack labels ที่สมบูรณ์ให้ฝึก supervised classifier โดยตรง. ระบบที่สร้างจึงใช้ protocol semantics และ normal profile แทนการพึ่ง anomaly score เดี่ยว.

แกนหลักของระบบ: Normal profile + TCP/MQTT rules + stream context + hard-negative exclusions = interpretable attack detector
DATA

Input

X_train.csv 100,000 rows, Normal reference. X_test.csv 10,000 rows สำหรับ inference.

MODEL

Core model

Deterministic rule engine. ไม่มี neural weights หรือ serialized checkpoint.

AUDIT

ML experiments

ExtraTrees, RandomForest, HistGradientBoosting และ IsolationForest ใช้จัดอันดับ/ตรวจ residual candidates.

RISK

Primary risk

Rare-normal, pseudo-label bias, stream leakage และ external domain shift.

คำว่า training ใน core system หมายถึงการสร้าง support sets จาก Normal train และการเลือก rule signatures จาก protocol knowledge. ไม่ใช่ gradient descent.

3. Data pipeline

CSV input | v Schema check + dtype inspection | +--> Normal profile / known shapes | +--> Protocol rules | +--> Frequency features + optional ML audit | v Context gate + hard-negative exclusion | v Binary attack decision + validation report

3.1 Load and preserve

train = pd.read_csv(TRAIN, low_memory=False)
test = pd.read_csv(TEST, low_memory=False)
  • รักษา row order และ Id identity
  • ไม่เปลี่ยน missing MQTT เป็นค่าทาง protocol จริง
  • เก็บ raw columns ก่อนสร้าง derived features

3.2 Build Normal profile

normal_windows = set(train["tcp.window_size"].dropna().unique())
normal_frame_windows = pd.MultiIndex.from_frame(
    train[["frame.len", "tcp.window_size"]]
)

profile นี้เป็น support set: ค่าหรือ pair เคยเกิดใน Normal train หรือไม่. ไม่ใช่ probability distribution เต็มรูปแบบ.

4. Core model: deterministic rule engine

Source หลักคือ outputs/predict_final_model.py. ระบบอ่าน train เพื่อสร้าง normal profile แล้ว apply Boolean rules กับ test.

4.1 Unseen TCP window

attack_stack = ~test["tcp.window_size"].isin(normal_windows)

ช่วยจับ attack capture ที่ใช้ TCP stack หรือ window value นอก Normal support. ต้องใช้ร่วมกับ rules อื่น เพราะ device ใหม่อาจมี window ใหม่แบบ benign.

4.2 Unseen frame/window pair

attack_packet_shape = (
    ~test_frame_windows.isin(normal_frame_windows)
    & test["mqtt.msgtype"].ne(3)
)

ใช้ interaction ของ frame length กับ window แทน field เดี่ยว และตัด PUBLISH ออกจาก generic novelty เพราะ payload diversity สูง.

4.3 Protocol signatures

syn_flood = (
    test["frame.len"].eq(54)
    & test["tcp.window_size"].eq(512)
    & test["tcp.flags.syn"].eq(1)
    & test["tcp.flags.ack"].eq(0)
)

dictionary_connect = (
    test["mqtt.msgtype"].eq(1)
    & test["mqtt.kalive"].isin([60, 3600])
)

invalid_subscription = test["mqtt.msgtype"].isin([8, 9])

publish_attack = (
    test["mqtt.msgtype"].eq(3)
    & test["mqtt.len"].isin([19, 44])
)

4.4 Rule union

labels = (
    attack_stack
    | attack_packet_shape
    | syn_flood
    | dictionary_connect
    | invalid_subscription
    | publish_attack
)

OR union เพิ่ม recall. Correction layer ด้านถัดไปทำหน้าที่รักษา precision.

5. Context และ hard negatives

5.1 PUBLISH context

labels |= (
    test["mqtt.msgtype"].eq(3)
    & test["tcp.window_size"].eq(256)
)

นี่คือ family-level context gate. ไม่ใช่การ label ทุก PUBLISH row; ต้องพิจารณา stream, packet family และ source evidence ร่วมกัน.

5.2 PINGRESP hard negative

pingresp_false_positive = (
    test["frame.len"].eq(56)
    & test["tcp.window_size"].eq(253)
    & test["mqtt.msgtype"].eq(13)
)
labels &= ~pingresp_false_positive
Precedence: positive rule union → exclusion rule → final label. Model confidence สูงไม่ override known Normal family.

5.3 Structural context

  • stream มี packet template ครบหรือไม่
  • packet อยู่ตำแหน่งใดใน sequence
  • มี packet ขาดหรือเกินจาก family หรือไม่
  • TCP flags/window/state transition สอดคล้องหรือไม่
  • candidate ชน Normal corpus หรือไม่

6. Feature engineering สำหรับ residual analysis

Core rules ใช้ raw protocol fields. สำหรับ residual ranking มีการเพิ่ม frequency features จาก work/analyze_candidates.py.

6.1 Frequency features

นับการเกิดของ packet, TCP shape, MQTT shape และ stream-packet shape ใน train/test:

train_hash = pd.util.hash_pandas_object(train[columns], index=False)
test_hash = pd.util.hash_pandas_object(test[columns], index=False)
train_counts = train_hash.value_counts()
test["packet_train_count"] = (
    test_hash.map(train_counts).fillna(0).astype("int32")
)

6.2 Excess and novelty

test[f"{prefix}_excess"] = (
    1 - normal_ratio * train_count / test_count
).clip(0, 1)

test["window_novel"] = (
    ~test["tcp.window_size"].isin(normal_windows)
)

normal_ratio=0.07138 เป็น heuristic prior. ห้ามเรียกค่า excess ว่า calibrated probability.

6.3 Candidate ranking score

candidate_score = (
    0.45 * packet_excess
    + 0.25 * tcp_excess
    + 0.20 * stream_packet_excess
    + 0.10 * rtt_novel
)

7.4 Why ML is not final decision

  1. pseudo positives มาจาก rule engine จึงเกิด circularity
  2. attack class coverage ไม่สมบูรณ์
  3. rare-normal ได้ probability สูง
  4. external data มี domain shift
  5. probability สูงไม่ใช่หลักฐาน causal attack context

8. Training, tuning และ validation

8.1 Train self-check

train_labels = attack_mask(train, train)
if int(train_labels.sum()) != 0:
    raise RuntimeError("Rules flagged normal training rows")

นี่คือ guardrail ว่ากฎไม่ flag Normal train แต่ไม่ใช่ proof ของ recall เพราะ train ไม่มี attack labels.

8.3 Validation layers

  1. Schema: row count, columns, Id uniqueness, binary labels
  2. Self-consistency: rule engine ไม่ flag Normal train
  3. Group validation: split ตาม capture/session ไม่ random packet
  4. Hard negatives: PINGRESP, normal PUBLISH, TCP control packets
  5. Source evidence: exact, shape, stream, RTT/full-feature support

False positives

  • rare PINGRESP
  • normal PUBLISH ที่ payload/window ใหม่
  • TCP window shift จาก device stack
  • packet shape ที่ดูเหมือน attack แต่ไม่มี flow context

False negatives

  • attack ที่ใช้ packet format เหมือน Normal
  • attack ที่เห็นจาก sequence/frequency มากกว่า field value
  • MQTT transaction ที่ต้องดูหลาย packet
  • burst behavior ที่ raw packet ไม่ต่างจาก Normal

10. Limitations และ next steps

ข้อจำกัด

  1. ไม่มี attack labels ใน train จึงวัด true recall ได้ไม่เต็มรูปแบบ
  2. rule coverage จำกัดตาม attack families ที่รู้จัก
  3. window และ packet lengths อาจเรียนรู้ environment มากกว่า behavior
  4. stream-based analysis ผูกกับ capture structure
  5. pseudo-label models มี circularity
  6. external PCAP มี domain shift
  7. random row split ให้ validation optimistic ได้

Production architecture

Protocol rules -> high-precision anchors Flow model -> context and recall Sequence model -> transaction semantics Hard-negative bank -> precision guardrail

Flow-level features

  • packet/byte count
  • direction ratio
  • inter-arrival distribution
  • burstiness
  • SYN/ACK/RESET transitions
  • MQTT transaction sequence

MQTT transaction features

CONNECT -> CONNACK -> PUBLISH
CONNECT -> CONNACK -> SUBSCRIBE -> SUBACK
PINGREQ -> PINGRESP

เป้าหมายต่อไปคือ group-held-out learning บน flow/session aggregates โดยยังคง protocol rules เป็น explainable anchors.
