01 / PROBLEM

โจทย์ไม่ได้ถามว่า packet แปลกไหม

โจทย์ถามว่า packet นี้เป็น Attack หรือเป็น Normal ที่พบไม่บ่อย

TRAIN
100,000แถว Normal ทั้งหมด

ไม่มี attack label ให้เรียน boundary โดยตรง.

TEST
10,000แถวสำหรับทำนาย

มีทั้ง Normal และ Attack; output ต้องเป็น Id,label.

METRIC
F1Precision + Recall

เพิ่ม positive แบบกว้างอาจทำคะแนนตก แม้ anomaly score สูง.

หลักคิด: อย่าใช้ “หายาก” แทน “โจมตี”. ต้องมี protocol meaning และ context.
อ้างอิงข้อมูลโจทย์: Kaggle competition

ย้ำความต่างระหว่าง rare-normal กับ attack. จุดนี้เป็นเหตุผลที่ broad novelty experiment ทำให้คะแนนตก.

02 / DATA MODEL

อ่านข้อมูลตามชั้นของ network

FRAME

Packet shape

frame.len, payload length, packet size.

ใช้หา shape ผิดปกติ แต่ไม่ตัดสินเดี่ยวๆ.

TCP

Transport state

tcp.flags, tcp.window_size, tcp.stream.

บอก state และความต่อเนื่องของ flow.

MQTT

Protocol meaning

mqtt.msgtype, mqtt.kalive, mqtt.len.

แยก CONNECT, SUBSCRIBE, PUBLISH, PINGRESP.

สมมติฐานที่ใช้

  • TCP/MQTT semantics ให้ precision anchor.
  • stream context ช่วยปิดช่องว่าง.
  • hard negative มีสิทธิ์ลบ candidate.

สิ่งที่ไม่ทำ

  • ไม่ label ทุก row ที่ rare.
  • ไม่อ้าง offline score เป็น Kaggle score.
  • ไม่อ้างอันดับหนึ่งโดยไม่มี leaderboard evidence.
Field reference: Wireshark MQTT fields

อธิบาย 3 ชั้น: frame, TCP, MQTT. Missing MQTT field อาจเป็น TCP control packet ไม่ใช่ข้อมูลเสีย.

04 / REAL CODE

โค้ดจริง 01: สร้าง rule mask

คัดจากไฟล์ production ใน repo. ไม่ใช่ pseudocode.

สิ่งที่กฎจับ

  • window ที่ไม่เคยอยู่ใน Normal
  • SYN flood signature
  • Dictionary CONNECT
  • Invalid subscription / Publish signature
Source: outputs/predict_final_model.py
outputs/predict_final_model.py
def attack_mask(train, test):
    normal_windows = set(
        train["tcp.window_size"].dropna().unique()
    )

    attack_stack = ~test["tcp.window_size"].isin(normal_windows)
    syn_flood = (
        test["frame.len"].eq(54)
        & test["tcp.window_size"].eq(512)
        & test["tcp.flags.syn"].eq(1)
        & test["tcp.flags.ack"].eq(0)
    )

    invalid_subscription = test["mqtt.msgtype"].isin([8, 9])
    publish_attack = (
        test["mqtt.msgtype"].eq(3)
        & test["mqtt.len"].isin([19, 44])
    )

เปิด code แล้วชี้ว่า rule ใช้ค่าจาก train สร้าง normal profile. ไม่ได้ train binary classifier จาก attack labels.

05 / REAL CODE

โค้ดจริง 02: เพิ่ม context แล้วลบ false positive

context gate + hard-negative exclusion
labels = attack_mask(train, test)

# Confirmed PUBLISH capture family
labels |= (
    test["mqtt.msgtype"].eq(3)
    & test["tcp.window_size"].eq(256)
)

pingresp_false_positive = (
    test["frame.len"].eq(56)
    & test["tcp.window_size"].eq(253)
    & test["mqtt.msgtype"].eq(13)
)
labels &= ~pingresp_false_positive
HARD NEGATIVE

PINGRESP ไม่ใช่ attack เพียงเพราะ rare

PUBLISH window 256
