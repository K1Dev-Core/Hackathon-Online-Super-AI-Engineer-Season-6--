02 / PROBLEM

ทำไมโจทย์นี้ยาก

  1. Train ไม่ได้ให้ attack labels
    ต้องสร้าง Normal profile แล้วหา deviation ที่มีความหมาย
  2. Rare-normal คล้าย attack
    packet แปลกหนึ่งแถวไม่เท่ากับการโจมตี
  3. Attack บางชนิดใช้ format ปกติ
    ความผิดปกติอยู่ที่ลำดับ, stream, frequency หรือ state
  4. F1 ลงโทษ false positive
    baseline สูงแล้ว candidate กว้างทำคะแนนตกเร็ว

โจทย์จริง

ไม่ใช่ “ทายว่าแถวนี้เป็น attack ไหม” อย่างเดียว

แต่คือ “แถวนี้เป็นส่วนหนึ่งของ attack family หรือเป็น rare-normal ที่ต้องปล่อยไว้”

ONE-CLASS + CONTEXT

หมายเหตุผู้บรรยาย

อธิบายความต่างจาก supervised classification: ไม่มี positive train labels ให้เรียน boundary โดยตรง. ต้องใช้ domain knowledge และการทดสอบแบบ controlled.

Rule union

เพิ่ม recall ด้วย OR หลายกฎ.

Exclusion

ตัด family ที่พิสูจน์ว่าเป็น normal.

Rollback

ทุก candidate เป็น delta แยกไฟล์. ย้อนกลับได้.

หมายเหตุผู้บรรยาย

เน้นว่า pipeline ไม่ได้ทำ random search. ทุกการเปลี่ยนมี hypothesis, changed Id list, checksum และผลจริง.

06 / RULE ENGINE

กฎหลัก: แปลง protocol meaning เป็น label

# high-precision anchors
attack_stack = ~test["tcp.window_size"].isin(normal_windows)
syn_flood = (
  test["frame.len"].eq(54)
  & test["tcp.window_size"].eq(512)
  & test["tcp.flags.syn"].eq(1)
  & test["tcp.flags.ack"].eq(0)
)
dictionary_connect = (
  test["mqtt.msgtype"].eq(1)
  & test["mqtt.kalive"].isin([60, 3600])
)
invalid_subscription = test["mqtt.msgtype"].isin([8, 9])

Anchors ที่ใช้

  • unseen TCP window
  • new packet shape
  • SYN flood signature
  • dictionary CONNECT
  • invalid subscription
  • PUBLISH length family
!

OR ช่วย recall แต่ OR กว้างเกินไปทำ FP. จึงต้องตามด้วย context gate และ exclusion.

หมายเหตุผู้บรรยาย

แสดง code จริงจาก deterministic model. อธิบายว่ากฎแต่ละตัวจับ attack family คนละแบบ.

14 / NEXT LEVEL

ถ้าจะชนะด้วยเครื่องแรงขึ้น ต้องเปลี่ยนหน่วยวิเคราะห์

Flow

direction, inter-arrival, burst, byte/packet rate

State

SYN/ACK transitions, reset ratio, session lifecycle

MQTT sequence

CONNECT → CONNACK → PUBLISH/SUBSCRIBE transition

Grouped validation

split by capture/session, not random packet

Experiment gate

candidate ต้องมี support, ไม่ชน normal corpus, ผ่าน hard negatives และคาด precision สูงกว่า F1 break-even.

Artifact discipline

immutable baseline, changed Ids, observed score, checksum, rollback path.
